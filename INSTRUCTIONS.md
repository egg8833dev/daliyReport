# 每日資訊日報 — 生成指引 v3

> v3 更新重點:在 v2 的 12 區基礎上,新增 5 個「能力養成類」區塊(系統設計、AI 工程實戰、深讀計畫、開發者效率工具、技術寫作與個人品牌),總計 17 區。能力養成類區塊的共同特徵:**有課綱、有進度、有累積**,每日推進並以進度註解追蹤,與資訊流區塊互補。
>
> v2 既有重點:①深度解析+快訊雙層結構 ②財經數據時效 fallback ③防重複規則(歷史/勵志/職涯/演算法)④金融科技區 ⑤連結驗證與持股外部化 ⑥每週素材回饋迴路

---

## 基本規則

- 語言:繁體中文為主,專業術語保留英文
- 每則資訊末尾附來源連結:`[→ 媒體名稱](實際文章URL)`
- 優先查詢一手來源(官方 Blog、原始論文、官方公告),避免純 SEO 整理文章
- 每個區塊末尾加 `📌 今日重點:一句話總結`
- 財經資訊標註「僅供參考,非投資建議」

### 深度分層結構

資訊流區塊(第 3、4、5、11、15 區)一律分為兩層:

1. **🔎 深度解析(每區 1–2 則)**:當日該領域最重要的主題,400–800 字完整說明,必含:
   - **背景脈絡**:這件事為什麼發生、之前的狀態是什麼
   - **技術/原理拆解**:不只報導「發生了什麼」,要解釋「怎麼運作」「為什麼這樣設計」
   - **對比與權衡**:與既有方案/競品/舊版本的差異,trade-off 是什麼
   - **【影響】**:對開發者/工程師的實際意義,具體到「你該做什麼/不用做什麼」
   - 有程式碼可展示時附最小可行範例(JavaScript 優先)
2. **⚡ 快訊(每區其餘則數)**:每則 1–2 句說明 + 【影響】一句話 + 來源連結

> 深度解析的選題標準:優先選「三個月後回看仍然重要」的結構性變化,而非單日話題性新聞。

### 能力養成類區塊共同規範(v3 新增)

適用於第 7(演算法)、8(系統設計)、9(AI 工程)、13(深讀計畫)區:

- **課綱輪替制**:依各區課綱依序推進,每篇報告末尾以 HTML 註解記錄進度,隔日從下一單元接續
- **內建知識為主**:這些屬穩定知識,原則上不需大量網頁搜尋,把搜尋額度留給資訊流區塊
- **程式碼須驗證**:所有可執行範例寫入前須於 sandbox 用 `node` 跑過(含邊界測試),並附測試案例與輸出
- **面試口述版**:每個主題末尾附 3–4 句「面試被問到時怎麼講」的口述示範
- **寫透優於寫多**:單一主題完整講清楚,勝過多主題淺嘗

### 抓取方式規範

- **GitHub Trending、Product Hunt、Hacker News 首頁**這類即時頁面,搜尋引擎常抓不到當日內容:優先直接 fetch 頁面或使用 RSS/API(HN 可用 `hnrss.org` 或 Algolia API)
- 官方 Blog 優先 fetch 其 RSS/Atom feed 確認「當日或近三日」是否有新文章
- 搜尋結果的日期必須核對,超過 7 天的內容不得標示為「今日動態」(深度解析與能力養成區的背景知識不在此限)

### 連結驗證規範

- 報告寫入前,對所有外部連結做可達性檢查(HTTP HEAD 或 GET,status < 400 視為有效)
- 失效或無法確認的連結,降級為「來源:媒體名稱(無直連)」,**嚴禁保留猜測拼湊的 URL**
- LeetCode 題目連結格式固定為 `https://leetcode.com/problems/<slug>/`,slug 必須與題名核對

### 2. 💹 財經與股市

來源:Bloomberg、Reuters Finance、玩股網、鉅亨網、Yahoo 股市、TWSE 臺灣證券交易所、期交所(台指期夜盤)

> **必須搜尋當下最新數據,不可沿用舊報告的數字。** 每次執行都要重新查詢,並在區塊開頭標註各項數據的「資料時間(台灣時間)」。所有內容標註「僅供參考,非投資建議」。

> **數據時效 fallback 規則(務必遵守):**
> - 生成時間若在美股收盤前(冬令收盤為台灣時間 05:00),美股數據標註為「盤中即時(截至 HH:MM)」,不得寫成收盤數據
> - 台指期夜盤交易至 05:00:若夜盤尚未收盤,標註「夜盤盤中」;若已收盤,標註「夜盤收盤」
> - 任何抓不到即時數據的項目,明確寫「暫無法取得即時報價,以下為最近可得數據(日期)」,不得用舊數字冒充今日數據

**固定版型(依序,每段用 `###` 子標題):**

1. **美股(標註盤中/收盤)**:道瓊、那斯達克、S&P 500、羅素 2000、費城半導體指數的最新點位與漲跌幅;簡述當日盤面驅動因素(2–3 句,點名具體事件而非「市場情緒」這類空話)。
2. **台股夜盤(台指期)**:即時/最後行情,說明相對日盤收盤的強弱與盤後氛圍,並註明與美股走勢的連動或背離。
3. **台股前一交易日(日盤收盤)**:加權指數(TAIEX)收盤點位與漲跌幅、成交金額;若外資買賣超數據可得,一併列出(來源:TWSE / Yahoo 股市)。
4. **持有個股報價**:讀取持股清單檔 `holdings.json`(格式見文末附錄);若檔案不存在,使用以下預設清單:
   - 台積電(2330)
   - 鴻海(2317)
   - 元大台灣50(0050)
   - 元大台灣50正2(00631L,槓桿型,須提醒波動約大盤 2 倍、控管風險)

   每檔查詢最新股價並標註日期;若 `holdings.json` 含成本價欄位,加列與成本價的相對表現。
5. **國內外財經重要資訊**:Fed / 央行政策與利率決議、匯率(至少含 USD/TWD)、重要財報、地緣與原物料(油價)等對市場有實質影響的消息 3–5 則。
6. **🔎 財經深度解析(每日 1 則)**:從當日財經消息中挑 1 個主題深入說明,400 字以上,包含歷史對照與多空兩面觀點。
7. **盤前注意事項 2–3 點**:結合上述資訊給出當日操作觀察重點(標註「僅供參考,非投資建議」)。

> **週末特別處理**:週六、週日改為「本週市場回顧」:整週漲跌幅、重要事件摘要、持股一週表現、下週財經行事曆。週末的深度解析改為「本週最重要的一件市場結構性事件」複盤。

#### 5-1 資安漏洞 CVE(分級呈現)

今日重要 CVE 依「與讀者相關程度」分三級,先讀最相關的,避免每天淹沒在無關高分漏洞裡:

- **🔴 直接相關**:命中讀者技術棧(.NET / Node.js / PostgreSQL / Docker / Nginx / K8s 生態 / Redis)。完整列出 CVE 編號、CVSS、影響版本範圍、修補版本、詳情連結,並附【自我檢查】。
- **🟡 值得知道**:廣泛影響的基礎設施元件(OpenSSL、glibc、curl、廣用函式庫等),即使非直接使用也可能透過依賴受影響。列出編號與一句話影響說明。
- **⚪ 一般知悉**:其餘高分或話題性 CVE,一句話帶過即可。

每則 🔴 / 🟡 末尾固定加:

> **【自我檢查】**:如何確認自己有沒有中招(檢查指令 / 版本比對方式)+ 緩解或修補步驟。
> 範例:某 Nginx CVE → 檢查:`nginx -v` 確認版本;緩解:升級至 `x.y.z` 或停用受影響模組。

每則說明:影響什麼、期限(何時生效)、遷移建議。

### 7. 🧮 演算法

**目標:每日 1–2 個主題,內容務求詳細完整(單一主題寫透優於多主題淺嘗)**

> **程式碼語言規範:本區塊所有範例程式碼一律使用 JavaScript(ES6+),不使用 Python 或其他語言。**

> **品質規範(務必遵守):**
> 1. **程式碼需經執行驗證**:每段範例程式碼寫入前須於 sandbox 用 `node` 實際跑過(含至少 2 組邊界測試,如空陣列、單元素、重複值),並在文中附上測試案例與預期輸出
